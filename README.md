# 📔 Forest Note - 毎朝開きたくなるライフログ

毎日の日記を記録し、AI が伴走する**ゲーミング習慣追跡アプリ**。森の成長・カード進化・実績解除で、継続を楽しくする体験を実現。

🌐 **本番デモ**: https://diary-calendar-mu.vercel.app  
📦 **バージョン**: v1.3.0 (ゲーミング完全版)  
🚀 **ステータス**: 安定版・本番運用可能  
✨ **新機能**: 森システム・AI記憶・成長レポート・カード進化・実績システム・カレンダー強化

---

## 🎯 コンセプト

**「毎朝開きたくなる」を実現する、AI伴走型習慣追跡日記アプリ**。

- **3分で完結する朝の記録フロー** → テキストペースト → AI自動質問・要約
- **AI 記憶**: 過去14日のデータから個人化された質問を生成
- **成長レポート**: 保存直後に、ユーザーの状態に寄り添う励ましメッセージ
- **3軸アドバイス**: 「今日・明日・今週」へのAIアドバイスを毎日提供
- **森の成長**: 習慣達成・連続記録・カード進化・実績解除で、ユーザーの庭の「森」が育つ
- **カード進化**: 日常の記録がゲーム要素を持つ「カード」に変わり、Lv1～100で進化・レアリティ上昇
- **実績システム**: 25個のバッジ・4段階の称号で成長を可視化
- **習慣のゲーミング化** → カレンダーで時系列確認 → 実績解除で達成感

## ✨ v1.3.0 実装済み機能

### 🏠 ホーム画面
- **🌲 森の成長表示** - リアルタイム成長度（Lv 0-100）、動物・季節変化
- **🎯 今日の習慣** - 6つのチェックボックス（pushups/squats/plank/run/reading/ai_learning）
- **📊 統計表示** - 連続日数・今日の達成数・気分スコア
- **🏆 実績パネル** - 最新3個の解除実績、装備称号表示
- **ボトムナビ** - ホーム・入力・カード・カレンダー・ランキング・実績

### ✍️ 日記入力フロー
- **テキストペースト** - Ctrl+V で貼り付け
- **リアルタイムプレビュー** - 入力中に解析結果を表示
- **自動解析** - FOREST_NOTE形式を自動パース

### 💭 AI 質問チャット（コンテキスト対応）
- **過去14日データ参照** - 前日の習慣・気分・エネルギー・睡眠パターンを検知
- **個人化された質問** - 習慣トレンド・連続記録・睡眠不足に応じて質問内容が自動調整
- **フォローアップ質問** - 最大3回までの深掘り
- **Demo AI** - API キー不要で動作

### 📝 成長レポート（保存直後）
- **寄り添う励まし** - 9パターンの励ましメッセージを状況別に生成
- **自動表示** - 日記保存直後に2.5秒表示、自動クローズ

### 💬 AIアドバイス（3軸）
- **今日・明日・今週** - 各軸で2-3行の寄り添うアドバイス
- **過去データ反映** - 習慣・気分・エネルギー・睡眠・ストリーク・カード進化・森の状態から判定
- **説教ではなく応援** - すべてのテンプレートが非処方的（supportive）

### 🌲 森システム
- **ビジュアル化** - SVG リアルタイムレンダリング（樹木1-3本・草・花・動物・季節変化）
- **成長トリガー** - 習慣達成数・連続記録・カード数から Lv 0-100 を計算
- **段階的な変化** - Level に応じて視覚要素が段階的に解放される

### 🎴 カード進化システム
- **EXP ポイント制** - 習慣達成（3 exp各）+ ストリーク（5-20）+ 気分（2-5）+ エネルギー（2-5）
- **Lv 1-100 進化** - Normal（Lv1-9）→ Rare（Lv10-29）→ Epic（Lv30-59）→ Legendary（Lv60-99）→ Mythic（Lv100）
- **進化時の報酬** - レアリティ上昇時に短いアニメーション表示・報酬メッセージ

### 📅 カレンダー強化
- **月別表示** - 全体俯瞰・日付クリックで詳細表示
- **日別詳細モーダル** - カード・習慣・気分・エネルギー・AI成長レポート・AIアドバイス・森状態を一画面で確認
- **背景色による気分可視化** - 記録した日付を気分レベルで色分け

### 🏆 実績システム
- **25個のバッジ** - ストリーク（5個）・習慣（3個）・カード進化（4個）・森成長（4個）・スペシャル（1個）
- **実績解除アニメーション** - 黄色グラデーション背景、バウンスイン、パーティクル効果
- **4段階の称号** - 駆け出し（5個）→ 成長中（15個）→ マスター（25個）→ 伝説（全個）
- **統計表示** - カテゴリ別集計・進捗率・次の目標表示

### 🎴 カード一覧画面
- **グリッド表示** - レスポンシブ対応（2～4列自動調整）
- **レアリティフィルター** - 進化度別表示
- **ソート機能** - 新しい順 / レアリティ順 / Lv順
- **フリップアニメーション** - カード表裏の3D回転
- **EXP バー表示** - 各カードの進化度を視覚化

### 🏆 ランキング画面
- **表彰台デザイン** - 1～3位を金・銀・銅で表示
- **自分の順位固定表示** - sticky で常時表示
- **タブ切り替え** - 今日・週間・月間・連続達成

### 🔐 デモモード（完全オフライン対応）
- **localStorage基盤** - Supabase 不要で即座に動作
- **自動サンプルデータ** - 初回アクセス時に14日分を生成
- **すべての機能対応** - 成長レポート・AIアドバイス・森・カード・実績すべてが動作

## 📚 AI 機能の使い方

### 典型的なユーザーフロー

```
1️⃣ テキスト入力 (/input/paste)
   ↓
   テキストを貼り付け → Forest Note 形式を自動解析
   
2️⃣ 💭 AI 質問チャット (/input/ai-questions)
   ↓
   5 つの質問に答える（気分・イベント・学び・体調・目標）
   各質問に 1～2 分程度
   
3️⃣ ✨ AI 要約生成 (/input/ai-summary)
   ↓
   AI が自動生成：タイトル・要約・タグ・属性
   気分・エネルギーを設定
   すべて編集可能
   
4️⃣ 🎴 カード確認 (/input/confirm)
   ↓
   AI 生成コンテンツが反映されたカードを確認
   カード表裏はフリップしてプレビュー
```

### Demo AI の特性

**初期状態（デフォルト）**
- API キー不要
- 固定ロジックで質問・タイトル・タグを生成
- 本番環境でそのまま動作
- 応答が即座（遅延なし）

**想定される出力例**
```
質問: 今日の朝、気分はいかがでしたか？ (1-5 段階)
答え: 4

AI 生成タイトル: "07 (月) - 今日の成長"
AI 生成タグ: ["日常", "学び", "成長", "感動"]
AI 属性: "mind" (メンタル)
```

## 🔧 デモモード ↔ 本番モード

### デモモード（デフォルト）
```bash
# .env.local
NEXT_PUBLIC_DEMO_MODE=true
NEXT_PUBLIC_SUPABASE_URL=https://placeholder.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=placeholder
NEXT_PUBLIC_AI_PROVIDER=demo
```

**特性**:
- ✅ Supabase不要
- ✅ localStorage にデータ保存
- ✅ 初回自動でサンプルデータ生成
- ✅ 仮ユーザーで自動ログイン
- ✅ Demo AI で即座に動作（API キー不要）

### 本番モード（Supabase有効 + 任意のAI）
```bash
# .env.local

# Supabase 設定
NEXT_PUBLIC_DEMO_MODE=false
NEXT_PUBLIC_SUPABASE_URL=your_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key

# AI Provider 設定（オプション）
NEXT_PUBLIC_AI_PROVIDER=demo              # "demo" | "openai" | "gemini"
NEXT_PUBLIC_AI_API_KEY=sk-...            # API キー（provider が "demo" の場合は不要）
NEXT_PUBLIC_AI_MODEL=gpt-3.5-turbo       # モデル名
```

**AI Provider の選択**:
- `demo` - 固定ロジック（推奨・API キー不要）
- `openai` - OpenAI API（v1.3 以降で実装予定）
- `gemini` - Google Gemini API（v1.3 以降で実装予定）

Supabase セットアップは [docs/SUPABASE_SETUP.md](./docs/SUPABASE_SETUP.md) を参照。
